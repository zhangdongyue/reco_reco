CrossMedia: 图像信息在阿里妈妈CTR预估系统中的应用

HIGHLIGHTS

CrossMedia项目通过图片对广告（Ad）和用户（User）建模，取得较原有ID类特征更好的离线效果。项目自十月底在阿里妈妈精准业务线单品主力广告位上线，实验桶相比对照桶（连续7天累计）点击率（CTR）提升7.5%，平均收益（RPM）提升4.7%。目前，在该系列广告位已经扩流超过70%。

除了业务效果，项目取得的技术进步更令人兴奋：
1. 告别了传统的使用现成图像特征加入预估网络的方法。而是将深层图像网络的后半部分加入CTR预估网络，同时参与训练。更加端到端的训练方式使得提取出的图像特征更贴近CTR预估的目标。也为最终实现从图像像素到CTR端到端训练作好了架构设计。
2. 在业界首次提出用图像信息对用户（行为）进行建模。我们将用户历史点击过的大量图片，引入到对用户的描述中，帮助描述用户兴趣。实验证明，通过图像对广告宝贝和用户同时建模，可以得到大幅提高图像带来的收益。
3. 用户的丰富行为带来了巨大的图片处理量，为应对海量训练图片数据带来的计算和存储挑战，我们首次提出了超越PS架构的新型架构--高级特征服务（Advanced Feature Service），从参数分布式架构升级为子模型分布式架构，支持复杂异构子模型的分布式存储、链接和联合计算。

背景

点击率（CTR）预估是广告/推荐系统中的核心任务，对于阿里妈妈展示广告按点击付费（CPC）的商业模式尤其如此。传统上关于CTR预估主要依赖历史反馈特征配合浅层模型（如逻辑回归）。近年来，CTR预估系统框架有了新进展，以ID特征+深度网络为基础的结构成为重要方向。阿里妈妈展示广告的CTR预估系统采用了其中的代表-- DIN度兴趣网络），除此之外还有Google的Deep&Wide模型，华为的DeepFM等。

然而，目前阿里妈妈展示广告CTR预估的特征体系中，以图片为代表的内容特征一直是缺位的。内容特征是相对于ID特征的概念，例如对于创意图，如果只记录图片的ID，则是ID特征；而如果描述了图片的像素、颜色直方图、属性等底层或高层特征，则是内容特征。一般认为，ID特征和内容特征是相辅相成的：前者长于记忆，而后者长于泛化。

作为展示广告的入口，图片的重要性显而易见，图片的吸引性直接关系到其点击率。因此，我们从图像内容入手，探索内容特征在CTR预估中的突出作用。可以预见，文字、视频等跨媒介的内容特征也可以通过完全类似的路径添加到系统。 这也是我们将项目命名为CrossMedia的原因。

事实上，已经有一些相关工作关注到图像特征在CTR预估中的作用，华先胜老师组的 DeepCTR 模型就是其中的先驱。我们的CrossMedia模型在此基础上进一步充实和扩展。众所周知，对广告（Ad）和用户（User）联合建模是个性化广告的基石。于是，我们尝试用图像对Ad和User同时建模，具体而言，我们在User的行为建模中加入其历史行为（这里用点击，也可以是收藏、购买等行为）的所有商品图片。

综合起来，CrossMedia系统包含了ID特征以及关于Ad和User的图像内容特征，示意图见图1。我们按以上功能设计仔细设计了模型结构和系统实现，将在后面两节涉及。

图1 联合训练ID特征和图像内容特征，图像特征同时建模Ad和User描述

图片对Ad和User行为建模

广告涉及到的图片很多，如商品主、副图，广告创意图。考虑到一次广告展现中，直接展现给用户的是其中的广告创意图，因此我们用所展示的广告创意图对广告建模，简记为Ad image。而对于用户来说，我们用用户的点击行为图片组对用户建模，简记为User images。用户行为很多，我们挑选出其过去两周的的典型行为（用停留时长等预测典型性）。经过筛选，平均每个用户有约31张User images描述。

模型设计

与通用方法一致，我们将CTR预估抽象成一个二分类问题，对于给定样本（一次广告展现），监督信息为点击/未点击。以此设计了一个全深度学习网络模型如下图所示：

图2 CrossMedia模型

整个CrossMedia模型分为特征表达模块和判别模块。在表达模块，不同的特征输入产出等长的特征向量。我们将表达模块中用于处理ID类特征、Ad图像、User图像的特征分别定义为Basicnet、**Adnet和Usernet。Basicnet为单层全连接，与主流网络（DIN等）Embedding层一致。在判别**模块，多种特征得到的向量拼接后，经过几层全连接，得到二分类输出。整个模型参数以分类误差为监督参与更新。

Adnet设计

如下图所示，考虑到系统性能和效率的平衡，我们将Adnet分为固定和可训练两部分，其中固定网络采用了成熟的VGG16网络的前端（截止FC6层），参数固定不参与训练。而可训练网络为三层全连接。这个看似简单的结构是我们尝试了1.Resnet等常见网络 2. VGG16经过淘宝图片分类FineTune(我们发现区别不大) 3. VGG16网络选用层数 4. 可训练网络结构调整 等多个变量最终确定的。

图3 Ad模型

Usernet设计

User描述与Ad描述的不同之处在于User行为图片是不确定多的，因此我们设计了一个Aggregator结构将每个User涉及的多个图片特征向量Pooling为一个描述向量。而每张图片的抽取向量与Adnet结构一致（但并不共享参数）。如下图所示：

图4 User模型

除了简单的Sum Pooling和Max Pooling外，我们尝试了近年来流行的Attentive Pooling的方式，即Pooling权重由User图片特征和其他特征预测得到。定性和定量实验表明，Attentive Pooling的方式通过赋予相关图片更高权重的方式，取得更好效果（见实验部分）。

训练系统实现

CrossMedia 最大的挑战来自于系统实现。图片是高密度信息源，即使经过VGG16固定网络的预提取，每张图也需要4096浮点数表示。同时，引入User表达大幅增加了系统涉及的图片数。经过筛选，平均每个User（两周内）有约31个点击行为，这就意味着，每个样本所涉及的User图片有31张，一个典型的训练Batch（Batchsize=3000）理论图片数高达10万张！虽然有一些合并样本的技巧，也是通常的分布式训练系统难以承受的。比如下面两种我们曾经考虑过的方案：

  1. 图片特征作为样本的一部分顺序写入。显然，图像特征的数量远超ID特征，这样的方式使得样本总量膨胀而无法存储。
  2. 建立起特征服务，维护一个分布式存储池，训练进程以KV方式访问。然而，由于图片数据量大，特征访问通信负载过高。

为解决上述问题，我们开发了分布式深度学习平台X-DeepLearining（XDL)，其中创新性地加入了“高级特征服务”，即带模型处理的特征服务。图5是它的系统框架。

在“高级特征服务”中，图像相关网络（即Adnet和**Usernet**的可训练部分）的预测和参数更新放在特征服务端完成，这样特征服务返回的特征是经过网络处理后即将参与Pooling的图像特征，根据设计，它的维度远小于原始特征，也大大降低了通信负载。

图5 基于“高级特征服务”的XDL系统框架

实践中，我们采用18天约40亿样本的训练集。在XDL平台40卡的GPU集群上，CrossMedia运行约10小时可完成两轮训练，达到了天级模型更新的要求。

离线效果

我们在阿里妈妈展示广告单品主力广告位验证了CrossMedia系统。出于严谨性，离线效果部分我们统一给出一个封闭集上的结果，该数据集来自今年七月连续18天作为训练集+第19天作为测试集，并去掉了交叉特征。在实际生产化中，我们发现虽然基线不同，但结论类似。为了论文表达明了，我们没有区分阿里妈妈体系的Creative ID和 Ad ID，也就是说，实验中的Ad ID实际上是指阿里妈妈体系的Creative ID。

我们采用Group AUC（GAUC）作为离线指标，GAUC是按User分组组内AUC的加权平均。在实践中，GAUC比AUC更贴近线上效果，也是我们一直使用的算法迭代指标。

在CrossMedia项目中，我们需要比较Ad端加图片特征的效果、Ad和User端同时加图片特征的效果。因此我们将所有特征分为三组：
1. basic ID: 其余所有ID类特征，如ad的店铺ID、类目ID等
2. ad ID 与 ad image: 分别是广告粒度的ID特征和图片内容特征，我们将比较二者单独作用和叠加效果
3. user(行为) IDs 与 user images: 分别是user 行为的商品ID和图片内容，我们同样比较二者单独作用和叠加的效果。注意这里的user IDs 不是用户本身的ID。

一些思考与展望

从直觉上看，图像（创意图）是展示广告的门面，对点击率影响很大。因此，我们曾多次尝试将图像信息加入CTR预估系统，但都不太成功。我们当时的做法是：用非监督（如BoW、FisherKernel等）或监督（如Alexnet以CTR为目标训练）的方法得到图像表达向量，再拼接到其他特征参与LR/MLR等浅层模型训练。现在看来，其中有两个问题：
1. 输入原始图像与CTR有很大的语义鸿沟，即通过像素推断其可能CTR需要依赖高度非线性变换，这样的推断任务即使对于人类也是很大挑战（人很难通过图像判断其CTR）。因此，借助Alexnet或BoW这样的浅层变换很难抽取出适合CTR预估任务的特征。
2. 我们只是简单地将图像看作特征，将（各种方式）提取到的图像特征放到其他特征集合体中参与训练，在这个过程中，特征提取与分类器训练是脱节的。

而CrossMedia很好地解决了上述问题：
1. CrossMedia网络使用了VGG16等更深的网络。VGG16网络是成熟的跨任务网络，我们虽然只训练了后几层，但正是由于前面的固定层提取的特征，使得后面CTR预估这样高度非线性任务变成了可能。
2. 由于深度学习技术的快速发展，我们能构建出包括了其他ID特征Embedding以及全连接网络的整体网络，这样可以将图像网络置于整体训练中，使得最终CTR监督可以回传更新图像网络。
除此之外，还获得了两方面的帮助：
3. 分布式GPU集群XDL开发成功，使得计算效率大大提高，我们能以更合理的方式组织样本，使图像网络更新变得非常频繁，有利于模型探索。
4. User 端引入图像特征大幅加大了算法收益，这在以前几次探索中都没有涉及。

这些都是CrossMedia在多次尝试后终于取得了正向效果的原因。项目下一步探索方向也是比较明确的：
在纵向上，我们将减少图像网络的固定部分，加入整体训练，朝End-to-End训练迈进。
在横向上，我们将以图像的实践为参照，扩展到广告的文字、视频等多种形式的内容特征，实现CrossMedia的初衷。
